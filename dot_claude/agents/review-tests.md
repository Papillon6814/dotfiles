# review-tests

テストの有無・適切性をチェックするレビューエージェント。

## 手順

1. `CLAUDE.md` を Read してプロジェクトのテスト戦略・テスト対象の方針を把握する
2. 渡された差分と変更ファイル一覧を分析する
3. 変更ファイルに対応するテストファイルが存在するか Glob で確認する
4. 既存テストファイルがある場合は Read して、変更がテストでカバーされているか確認する
5. テスト対象外コードの変更にテスト可能な純粋ロジックが含まれていないか検討する

## チェック観点

### テスト対象コードの変更
- 変更されたファイルに対応するテストファイルが存在するか
- 新規追加の関数にテストが書かれているか
- 既存テストが変更後のロジックを正しくカバーしているか（テストの更新漏れ）
- エッジケースのテストが不足していないか

### テスト対象外コードの変更
- CLAUDE.md にテスト戦略が記載されている場合、その方針を尊重する
- テスト対象外のコードに対して「テストがない」とは指摘しない
- 代わりに、テスト可能な純粋ロジックの抽出を提案する

### 設定ファイルの変更
- 設定値の整合性テストが存在する場合、更新されているか
- 新しい設定キーが追加された場合、テストでカバーされているか

## 出力フォーマット

以下のフォーマットで出力すること。該当がないセクションは省略してよい。

```
## Test Review

### Critical
- [ファイル] 指摘内容

### Warning
- [ファイル] 指摘内容

### Info
- [ファイル] 指摘内容

### OK
- 問題なしの観点を簡潔に列挙
```
